### 2.3 版本兼容性说明

- **MyBatis-Plus 3.5.9** 要求 `mybatis-plus-spring-boot3-starter`（非 `mybatis-plus-boot-starter`），明确支持 Spring Boot 3.x。
- **Knife4j 4.5.0** 必须用 `knife4j-openapi3-jakarta-spring-boot-starter`，Spring Boot 3.x 基于 Jakarta EE，不是 javax。
- **Druid 1.2.23** 同理，Spring Boot 3 用 `druid-spring-boot-3-starter`。
- **MySQL Connector** 包名已从 `mysql:mysql-connector-java` 变更为 `com.mysql:mysql-connector-j`。
- **jjwt 0.12.6** 三包必须同版本：`jjwt-api` / `jjwt-impl` / `jjwt-jackson`。

## 六、版本冲突排查速查表

| 常见冲突 | 原因 | 解决方案 |
|---------|------|---------|
| `ClassNotFoundException: javax.servlet.*` | 用了 javax 依赖 | Spring Boot 3.x 全部改 `jakarta` 包 |
| `NoClassDefFoundError: io.jsonwebtoken.*` | jjwt 三包版本不一致 | `jjwt-api/impl/jackson` 必须同一版本 |
| MyBatis-Plus 启动报错 | 引了 `mybatis-plus-boot-starter` | 改用 `mybatis-plus-spring-boot3-starter` |
| Knife4j 文档页 404 | 用了 javax 版 | 改用 `knife4j-openapi3-jakarta-spring-boot-starter` |
| `NoSuchMethodError` on StringUtils | Hutool 版本过旧 | 升级至 5.8.34 |
| Element Plus 图标不显示 | `@element-plus/icons-vue` 版本过旧 | 升级至 2.3.1 |
| Vite 启动报错 ESM 相关 | Node 版本过低 | 升级 Node ≥ 18，推荐 20 LTS |
| langchain 导入报错 | Python 版本不匹配 | Python ≥ 3.10，当前 3.13 |
| pydantic 报错 | pydantic v1/v2 混用 | 统一用 pydantic ≥ 2.0，langchain 0.3 已适配 |

## 七、快速初始化命令

### Java 后端
```bash
# 确保 JDK 21 已安装
java -version
# 使用 Maven Wrapper 构建
./mvnw clean install -DskipTests
```

### 管理端/前台端
```bash
npm create vite@6 hotel-backend-front-manage -- --template vue-ts
cd hotel-backend-front-manage
npm install vue@3.5.13 element-plus@2.9.1 @element-plus/icons-vue@2.3.1 \
  vue-router@4.5.0 pinia@2.3.0 pinia-plugin-persistedstate@4.2.0 \
  axios@1.7.9 dayjs@1.11.13 echarts@5.5.1 vue-echarts@7.0.3 sass@1.83.0
```

### Python 智能服务
```bash
# 确保 Python 3.13+ 已安装
python --version
# 安装 uv（如未安装）
pip install uv
# 创建虚拟环境并安装依赖
cd AI_hotel/ai-service
uv venv
uv pip install -e .
```
